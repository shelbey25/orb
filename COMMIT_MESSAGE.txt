# Setup Express + PostgreSQL + Prisma

## Summary
I've set up a complete backend infrastructure for your Orb application with Express, PostgreSQL, and Prisma, ready for Railway deployment.

## What's Included

### Backend Server (`server.ts`)
- Express API with full CRUD operations for students
- Endpoints: POST/GET/PUT/DELETE `/api/students`
- CORS enabled, JSON parsing middleware
- Graceful error handling
- Proper shutdown on SIGINT

### Database Configuration
- Switched from SQLite to PostgreSQL
- Prisma ORM setup with Student model
- Student fields: id, email (unique), name, archetypes (JSON), timestamps
- Ready for Railway PostgreSQL plugin

### Deployment Ready
- `railway.toml` configuration file
- Environment variables set up in `.env.local`
- Migration guide for production
- All dependencies installed

### Documentation
- `BACKEND_SETUP.md` - Complete setup guide
- `SETUP.md` - Quick start instructions
- `MIGRATIONS.md` - Database migration workflow
- `test-api.ts` - API test script

## Quick Start

1. **Local Development:**
   ```bash
   createdb orb_dev
   DATABASE_URL="postgresql://localhost/orb_dev" npm run db:push
   npm run server:dev
   ```

2. **Test API:**
   ```bash
   npx ts-node test-api.ts
   ```

3. **Deploy to Railway:**
   - Push to GitHub
   - Connect to Railway
   - Add PostgreSQL plugin
   - Done!
